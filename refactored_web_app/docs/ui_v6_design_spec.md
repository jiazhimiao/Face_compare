# 5020 UI 升级设计说明

本说明用于把 5020 版本从“代码微调”切换为“设计稿驱动”的落地方式，确保后续迭代先有结构和规则，再做样式与开发。

## 1. 页面结构

- 左侧导航区：版本回溯、设计升级摘要、Token 与服务信息。
- 顶部栏：工作区定位、版本动作入口、接口元信息。
- 中央工作区：按功能拆分的工作卡片，每张卡片包含上传区、结果层、媒体层、JSON 层。
- 右侧摘要区：最近一次处理结果、设计原则、验证清单。

## 2. 视觉系统

- 主色：`#2458D5`
- 成功色：`#1C986E`
- 警示色：`#C38818`
- 风险色：`#D6485B`
- 背景色：`#F3F6FB`
- 圆角体系：`14 / 18 / 24 / 32`
- 阴影体系：轻卡片阴影 + 顶层容器阴影

## 3. 结果展示层级

- 第一层：状态 Banner，先告诉用户是否成功、是否通过、是否命中。
- 第二层：指标卡片，集中展示相似度、阈值、人脸数、命中对象等核心字段。
- 第三层：结果图与原图切换，支持演示与问题回看。
- 第四层：原始 JSON，保留联调与排障能力。

## 4. 交互状态

- 空态：清晰告诉用户当前需要上传什么。
- 加载态：展示处理中 Banner 与骨架感占位。
- 成功态：状态色、摘要、图片与 JSON 同步更新。
- 错误态：保留错误文案与 JSON，便于排查。

## 5. 结果图组件要求

- 图内必须同时表达标题、结论状态、关键指标和图像区域。
- 单图与双图使用统一视觉语言。
- 中文标题与固定说明文案优先使用稳定的字体与 Unicode 文案策略。

## 6. 组件化要求

- 按钮、胶囊标签、状态徽标、字段输入、结果卡、指标卡、图片卡片、JSON 面板应复用统一样式变量。
- 新版本新增组件时，优先复用变量与容器样式，不直接新增散乱颜色。

## 7. 版本策略

- 5006 / 5009 / 5014 为冻结回溯版本，不再直接修改。
- 新实验、新设计、新交互均在 5020 及后续独立版本中推进。
- 后续如进入新一轮改版，应继续新增版本入口，保留历史稳定回溯路径。
